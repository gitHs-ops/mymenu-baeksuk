# 🍜 식당 QR 오더 테이블 앱

중국집 메뉴를 기반으로 한 QR 코드 주문 시스템입니다.

## 📋 기능

### 고객용 주문 앱 (index.html)
- ✅ 메뉴 카테고리별 분류 (탕수육, 요리, 면, 밥, 세트메뉴, 추가)
- ✅ 장바구니 기능
- ✅ 수량 조절 (증가/감소)
- ✅ 실시간 총액 계산
- ✅ 주문 완료 확인
- ✅ 테이블 번호 자동 할당
- ✅ 로컬 스토리지 저장 (새로고침 시 장바구니 유지)
- ✅ 모바일 반응형 디자인

### 업주용 관리 앱 (admin.html)
- ✅ 실시간 주문 수신 및 알림
- ✅ 주문 상태 관리 (대기중/조리중/완료)
- ✅ 통계 대시보드 (총 주문, 매출 등)
- ✅ 주문 필터링 및 검색
- ✅ 알림음 설정
- ✅ 주문 상세 보기
- ✅ 완료 주문 일괄 삭제

## 🚀 사용 방법

### 고객용 (손님)
1. 웹 브라우저에서 `index.html` 파일을 엽니다
2. 원하는 메뉴를 선택하고 "담기" 버튼을 클릭합니다
3. 우측 하단의 장바구니 버튼을 클릭하여 주문 내역을 확인합니다
4. 수량을 조절하거나 항목을 삭제할 수 있습니다
5. "주문하기" 버튼을 클릭하여 주문을 완료합니다

### 업주용 (관리자)
1. 웹 브라우저에서 `admin.html` 파일을 엽니다
2. 새 주문이 들어오면 자동으로 알림이 표시됩니다
3. "접수" 버튼을 클릭하여 주문을 조리중으로 변경합니다
4. 조리가 완료되면 "완료" 버튼을 클릭합니다
5. 필터 탭으로 주문 상태별로 확인할 수 있습니다
6. 우측 하단의 🔔 버튼으로 알림음을 켜고 끌 수 있습니다

## 📱 QR 코드 생성 방법

### 1. QR 코드 생성기 사용
1. 브라우저에서 `qr-generator.html` 파일을 엽니다
2. 테이블 번호를 입력합니다 (1-100)
3. 주문 페이지 URL을 입력합니다 (자동으로 현재 경로가 입력됨)
4. "QR 코드 생성" 버튼을 클릭합니다
5. 생성된 QR 코드를 다운로드하거나 인쇄합니다
6. 각 테이블에 QR 코드를 부착합니다

### 2. QR 코드 작동 방식
- QR 코드를 스캔하면 `index.html?table=1` 형식의 URL로 이동
- 테이블 번호가 자동으로 설정됨
- 손님이 해당 테이블에서 주문 가능

### 3. 온라인 QR 생성 서비스 (대안)
실제 서비스 배포 시:
1. 웹 서버에 파일 업로드
2. https://yourrestaurant.com/order?table=1 형식의 URL 생성
3. 온라인 QR 생성 서비스 이용:
   - https://www.qr-code-generator.com/
   - https://www.the-qrcode-generator.com/
   - https://goqr.me/

## 🔄 작동 원리

1. **고객이 주문**: index.html에서 주문 → localStorage에 저장
2. **실시간 전송**: admin.html이 3초마다 localStorage 확인
3. **주문 수신**: 새 주문 발견 시 알림음 + 브라우저 알림

## 🖥️ 포스(POS) 시스템 연동

### 현재 버전 (프로토타입)
현재 이 시스템은 **LocalStorage 기반 프로토타입**으로, 포스기와 직접 연동되지 않습니다.
- 브라우저 내에서만 작동
- 실제 포스기 연동 불가
- 데모 및 테스트용

### 포스기 연동을 위한 개선 방안

#### 1. 백엔드 서버 구축 필요
포스기 연동을 위해서는 중간 서버가 필요합니다:

```
[손님 앱] → [웹 서버] → [포스 시스템]
   ↓           ↓            ↓
주문 생성   주문 처리    매출 관리
```

**필요한 기술:**
- Node.js + Express 또는 Python + Flask/Django
- 데이터베이스 (MySQL, PostgreSQL, MongoDB)
- WebSocket 또는 Server-Sent Events (실시간 통신)
- REST API 또는 GraphQL

#### 2. 포스 시스템 연동 방법

**A. API 연동 (권장)**
대부분의 현대 포스 시스템은 API를 제공합니다:
- 포스피 (POSFI)
- 오케이포스 (OK POS)
- 토스페이먼츠 POS
- 배달의민족 사장님광장

**연동 절차:**
1. 포스 업체에 API 사용 신청
2. API 키 발급 받기
3. 백엔드 서버에서 포스 API 호출
4. 주문 데이터 자동 전송

**B. 프린터 연동**
API가 없는 경우:
- 주방 프린터로 주문서 자동 출력
- ESC/POS 프로토콜 사용
- USB 또는 네트워크 프린터 연결

**C. 화면 연동**
- 포스 화면에 웹 브라우저 띄우기
- admin.html을 포스 PC에서 실행
- 수동으로 포스에 입력

#### 3. 실제 구현 예시 (Node.js)

```javascript
// server.js (백엔드 서버)
const express = require('express');
const app = express();

// 주문 수신
app.post('/api/orders', async (req, res) => {
    const order = req.body;
    
    // 1. 데이터베이스에 저장
    await saveOrderToDB(order);
    
    // 2. 포스 시스템에 전송
    await sendToPOS(order);
    
    // 3. 주방 프린터로 출력
    await printToKitchen(order);
    
    res.json({ success: true });
});

// 포스 API 호출 예시
async function sendToPOS(order) {
    const response = await fetch('https://pos-api.example.com/orders', {
        method: 'POST',
        headers: {
            'Authorization': 'Bearer YOUR_API_KEY',
            'Content-Type': 'application/json'
        },
        body: JSON.stringify({
            tableNumber: order.tableNumber,
            items: order.items,
            total: order.total
        })
    });
    return response.json();
}
```
